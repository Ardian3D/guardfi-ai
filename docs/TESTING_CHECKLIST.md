# GuardFi AI — Testing Checklist

## Automated

- [ ] **Frontend unit tests** — `cd apps/web && npm test` (Vitest: risk engine, AI report fallback/parser/sanitizer, HSP mock/storage/config/service, dashboard aggregation).
- [ ] **Frontend build** — `cd apps/web && npm run build` (exit 0).
- [ ] **Contract compile** — `cd apps/contracts && npm run compile`.
- [ ] **Contract tests** — `cd apps/contracts && npm test` (`submitReport`, `getReport`, indexing, validation reverts).

## Manual — app flow

- [ ] **Wallet connect** — Launch App → connect → land on Dashboard.
- [ ] **Wrong network** — on a non-177 chain, NetworkBadge shows "Wrong Network"; `/connect` prompts to switch; WalletButton shows "Wrong Network".
- [ ] **Scan demo target** — Sample Target chip → Analyze → score + indicators + demo badge.
- [ ] **Scan real target** — paste a HashKey Chain contract → live bytecode/ERC-20/proxy/selector read.
- [ ] **Scan invalid address** — malformed input rejected client-side; non-contract address returns a clear "not a contract" error.
- [ ] **AI report (fallback)** — with `DEEPSEEK_API_KEY` empty → "Fallback Report" badge, all sections present.
- [ ] **AI report (DeepSeek)** — with a valid key → "Generated by DeepSeek" badge.
- [ ] **Unlock premium (mock)** — `NEXT_PUBLIC_HSP_MODE=mock` → modal shows "Mock HSP Mode" → Unlock → Created→Pending→Settled → report unlocked.
- [ ] **Reports locked/unlocked** — `/reports/[id]` locked before unlock, full AI report after unlock (per wallet).
- [ ] **Save proof on-chain** — with `NEXT_PUBLIC_REGISTRY_ADDRESS` set + HSK for gas → wallet signs → txHash + reportId shown.
- [ ] **Registry missing** — with address unset → "Registry contract address is not configured." warning.
- [ ] **User rejects tx** — reject in wallet → clear "Transaction was rejected" message, no crash.
- [ ] **Verify report ID** — `/verify` (no wallet) → enter Report ID `1` → reporter/target/score/hash/metadata/timestamp shown; optional hash match.
- [ ] **Verify not found** — unknown ID → "Report not found."
- [ ] **Dashboard activity** — scans, AI reports, on-chain proofs, premium unlocks reflected; empty state on fresh browser.
- [ ] **HSP real not configured** — `NEXT_PUBLIC_HSP_MODE` not `mock` → unlock surfaces "HSP adapter is not configured."

## Manual — responsive

- [ ] Landing page (mobile) — hero stacks, tech stack readable, footer intact.
- [ ] Dashboard (mobile) — summary cards wrap, table scrolls horizontally (no overflow break).
- [ ] Scan + Scan result (mobile) — form and result stack cleanly.
- [ ] Report detail (mobile) — sections stack; on-chain proof panel readable.
- [ ] Verify page (mobile) — form + result stack.

## Wording safety

- [ ] Grep returns **no forbidden claims** in UI (only disclaimers/guardrails/sanitizer):
  ```bash
  grep -ri "GuardToken\|guaranteed safe\|certified audit\|formal audit\|financial advice" apps
  ```
- [ ] No project token anywhere.
- [ ] Disclaimer "not a formal audit or financial advice" appears on scan result, report, verify, and footer.

## Routes

- [ ] Protected: `/dashboard`, `/scan`, `/scan/[id]`, `/reports/[id]`, `/target/[address]` redirect to `/connect?redirect=<path>` when disconnected/wrong network.
- [ ] Public: `/`, `/connect`, `/verify`, `/docs` + sub-pages.
